# OSAI Paper Analysis Pipeline

[![DOI](https://sandbox.zenodo.org/badge/1160122914.svg)](https://handle.test.datacite.org/10.5072/zenodo.469690)

This project was developed as part of the **Open Science and AI course project**.

It implements a reproducible pipeline for analysing scientific papers using **GROBID** and Python. The system processes research articles in PDF format and extracts structured information such as abstracts, figures and external links.

The project demonstrates key principles of **open science and reproducible research workflows**.

---

# Project Goals

The objective of this project is to:

- Automatically analyse scientific papers using text-mining tools
- Extract structured information from PDF documents
- Produce visualisations based on extracted data
- Demonstrate reproducible scientific workflows
- Follow open science best practices (documentation, environment reproducibility, metadata)

---

# Data

The analysis is performed on **10 open-access research papers** downloaded from **arXiv**.

These papers are stored in the repository under:

Each PDF is processed using the **GROBID full-text parsing API**, which converts the document into structured **TEI XML** format.

# Pipeline Overview

The analysis pipeline performs the following steps:

1. Each PDF is sent to the **GROBID API** using the `processFulltextDocument` endpoint.
2. GROBID converts the PDF into **TEI XML** format.
3. The XML is parsed to extract:
   - Abstract text
   - Number of figures
   - External links
4. The extracted information is aggregated and exported as visualisations and structured datasets.

--platform linux/amd64
-e JAVA_TOOL_OPTIONS="-XX:-UseContainerSupport"
-p 8070:8070

# Running the Pipeline with Docker

The project can also be executed using Docker for improved reproducibility.

## Build the container


docker build -t osai-project .


## Run the pipeline


docker run --rm
-e GROBID_URL=http://host.docker.internal:8070/api/processFulltextDocument

-v "$(pwd)/papers:/app/papers"
-v "$(pwd)/outputs:/app/outputs"
-v "$(pwd)/data:/app/data"
osai-project


This command will run the analysis and store the results in the local repository folders.

# Validation of Results

The extracted results were validated through manual inspection.

### Abstract extraction

Extracted abstracts were compared with the original PDFs for several papers to ensure that the parsing was correct.

### Figure detection

The number of figures was obtained by counting `<figure>` tags in the TEI XML produced by GROBID.

To validate this process, several papers were manually inspected and the detected figures were compared with the figures visible in the original PDFs.

### Link extraction

Extracted URLs were manually checked to confirm that they correspond to references or external resources present in the original papers.

Validation was performed by comparing the TEI XML output with the original PDF documents.

---

# Limitations

Several limitations are inherent to PDF parsing and automated document analysis:

- GROBID may incorrectly detect figures depending on the structure of the PDF
- Some URLs may be truncated if they are split across multiple lines
- Certain references may not be recognised as links
- Abstract extraction may fail for poorly formatted PDFs

Despite these limitations, the pipeline performs reliably for most modern scientific articles.
